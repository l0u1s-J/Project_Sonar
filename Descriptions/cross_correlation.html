Cross correlation :

the cross correlation of two discrete signals can be calculated with the following formula :
But in our case the signals are finite, the sent one going from 1 to 2879 and the recieved one going from 1 to 4319 (60 ms + 30 ms).
What we are interested in is the maximum of the cross correlation, which means the delay of the signal between sending and recieving. So we have to fill the send signal with zeros to have signals that have the same length
So in our case :
this can be done in a double for-loop like this:
n = 0:4319;
x1 = 0.84.^n; % example function, impulse like
x2 = circshift(x1,1392); % same function, but shifted by 1392 steps (29 ms for our ADC) to the right
r=zeros(1,8639); % declaration of the cross correlation function with zeros (important for the calculation)
for k=-4319:4319
for m=1:4319
if (m-k)<4321 && (m-k)>0
r(k+4320)=r(k+4320)+x2(m)*x1(m-k); % k+4320 the r vector goes from 1 to 8640 (no negative index)
end
end
end
%Illustration
t=-4319:4319;
subplot(3,1,1)
plot(n,x1)
title("x1(n)")
xlim([-1000 3000])
subplot(3,1,2)
plot(n,x2)
title("x2(n)")
xlim([-1000 3000])
subplot(3,1,3)
plot(t,r)
title("cross correlation betwenn x1 and x2")
As you see this code can very easily be implemented in C.

Distance measuring :

The first step to measure the distance is to find out the maximum of the cross correlation between sent and recieved signal.
For this, an algorithm can be directly implemented in the cross correlation double loop:
r=zeros(1,8639);
max_k=0;
for k=-4319:4319
for m=1:4320
if (m-k)<4321 && (m-k)>0
r(k+4320)=r(k+4320)+x2(m)*x1(m-k);
end
end
if k+4320>1 % starting at the second value
if r(k+4320) > r(k+4319) % compare the momentan value to the last value
max_k=k; % save the bigger value
end
end
end
Now that we have the max step k, we have first to calculate the time:
time = max_k*20.833e-6; % max_k * 1/fs
and then multiply it with the speed of sound to get the total distance gone by the sound, and then divide it by 2 to get the distance to the object:
distance = (340*time)/2
distance = 4.9299
Problem is the cross correlation vector r takes a lot of memory : 32 bit * 8639 = 276 kb
But we see here that we don't need the whole vector to calculate the distance, but only the momentan and the last value. So we can reduce the code to:
r_max = 0;
for k=-4319:4319
r_new=0;
for m=1:4320
if (m-k)<4321 && (m-k)>0
r_new=r_new+x2(m)*x1(m-k);
end
end
if r_new > r_max
r_max=r_new;
max_k=k;
end
end
time = max_k*20.833e-6;
distance = (340*time)/2
distance = 4.9299
So now the processor has to do the same amount of operations, but it does not have to save the 5759 32bit values of r, which is an improvement.
We can now implement directly this code in C to a SWI in our program.
